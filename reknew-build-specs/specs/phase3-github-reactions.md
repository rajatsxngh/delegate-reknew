# Phase 3: GitHub PR Integration + Reactions Engine

## Goal
Close the automation loop. When an agent completes work, automatically push to GitHub, create a PR, monitor CI, and handle failures/reviews by re-prompting the agent. The pipeline becomes fully autonomous: issue → code → PR → CI → merge.

## Prerequisites
- Phase 2 complete (parallel agents working with adapter interface)
- GitHub personal access token with `repo` scope
- A test repo with GitHub Actions configured (sp-enablers-slayer has this)

```python
class GitHubAPI:
    """Centralized GitHub API access.

    All GitHub operations go through this class. It handles:
    - Authentication (token from env or config)
    - Rate limit tracking and warnings
    - Consistent error handling
    """

    def __init__(self, token: str | None = None):
        """Initialize. Token from param, config, or GITHUB_TOKEN env var."""

    @property
    def rate_limit_remaining(self) -> int:
        """Current remaining API requests before rate limit."""

    def get_repo(self, repo_name: str):
        """Get a PyGithub Repository object."""

    def get_issue(self, repo_name: str, issue_number: int):
        """Get a PyGithub Issue object."""

    def get_pull(self, repo_name: str, pr_number: int):
        """Get a PyGithub PullRequest object."""
```

### 3. reknew/github/pr_manager.py

**Purpose:** Create and manage pull requests on GitHub.

```python
@dataclass
class PRResult:
    pr_number: int
    pr_url: str
    branch: str
    title: str

@dataclass
class CIStatus:
    overall: str               # "passed", "failed", "pending"
    checks: list[dict]         # [{name, state, description, url}]
    failed_checks: list[dict]  # subset of checks where state == "failure"

@dataclass
class ReviewComment:
    reviewer: str
    body: str
    state: str                 # "APPROVED", "CHANGES_REQUESTED", "COMMENTED"
    path: str | None           # file path for inline comments
    line: int | None           # line number for inline comments
    created_at: str

class PRManager:
    """Creates PRs and monitors their status on GitHub."""

    def __init__(self, github_api: GitHubAPI, config: ReknewConfig):
        self.api = github_api
        self.config = config

    def push_branch(self, repo_path: str, branch: str) -> None:
        """Push a local branch to GitHub origin.

        Only the daemon calls this — agents never push.

        Steps:
        1. git push origin {branch}
        2. If push fails (remote branch exists), force push with lease:
           git push --force-with-lease origin {branch}

        Raises:
            subprocess.CalledProcessError: if push fails
        """

    def create_pr(
        self, repo_name: str, branch: str, base: str,
        title: str, body: str, task_id: str, issue_number: int | None = None
    ) -> PRResult:
        """Create a pull request on GitHub.

        PR title: "{config.github.pr_prefix} {title}"
        PR body includes:
        - Task ID
        - Link to original issue (if issue_number provided)
        - Auto-generated summary of changes
        - "Generated by ReKnew AI-SDLC pipeline" footer

        If config.github.auto_label_prs is True, adds "reknew-ai" label.

        Returns: PRResult with pr_number and url
        Raises: github.GithubException if PR creation fails
        """

    def get_ci_status(self, repo_name: str, pr_number: int) -> CIStatus:
        """Check CI status on a PR.

        Combines GitHub Status API and Check Runs API:
        1. Get latest commit on the PR
        2. Fetch commit statuses (older CI systems)
        3. Fetch check runs (GitHub Actions, modern CI)
        4. Determine overall status:
           - Any failed → "failed"
           - All passed → "passed"
           - Otherwise → "pending"

        Returns: CIStatus with overall and per-check details
        """

    def get_review_comments(self, repo_name: str, pr_number: int) -> list[ReviewComment]:
        """Get review comments requesting changes.

        Fetches:
        1. PR reviews (APPROVED, CHANGES_REQUESTED, COMMENTED)
        2. PR review comments (inline, with file path and line number)

        Only returns comments from CHANGES_REQUESTED reviews
        and any inline comments, not approvals.

        Returns: list of ReviewComment objects
        """

    def merge_pr(self, repo_name: str, pr_number: int, method: str = "squash") -> bool:
        """Merge a PR via GitHub API.

        Args:
            method: "merge", "squash", or "rebase"

        Returns: True if merged, False if merge failed (conflicts, etc.)
        """

    def close_pr(self, repo_name: str, pr_number: int, comment: str = "") -> None:
        """Close a PR without merging (e.g., after max retries exhausted)."""

    def format_ci_errors(self, ci_status: CIStatus) -> str:
        """Format CI failure details into a prompt for the agent.

        Output format:
        CI failed on your PR. Please fix the following errors:

        ❌ Check: "pytest"
        Status: failure
        Details: 3 tests failed in test_models.py
        Log URL: https://github.com/...

        ❌ Check: "lint"
        Status: failure
        Details: 5 lint errors found
        Log URL: https://github.com/...

        Please fix these issues, commit, and push. The CI will re-run automatically.
        """

    def format_review_comments(self, comments: list[ReviewComment]) -> str:
        """Format review comments into a prompt for the agent.

        Output format:
        Code review requested changes on your PR. Please address:

        📝 Reviewer: john-doe
        Comment: "This function should handle the case where the input is empty"

        📝 Reviewer: jane-smith (on src/models.py, line 42)
        Comment: "Consider using a dataclass here instead of a dict"

        Please address all comments, commit, and push.
        """
```

### 4. reknew/github/webhook_listener.py

**Purpose:** Listen for GitHub webhook events (CI status, reviews). Falls back to polling.

```python
class WebhookListener:
    """Listens for GitHub events via webhooks or polling.

    Mode is determined by config.github.mode:
    - "webhook": FastAPI endpoint receives POST events from GitHub
    - "poll": periodically check PR status via API

    Events emitted:
    - "ci-completed": CI finished (pass or fail) on a PR
    - "review-submitted": someone submitted a review on a PR
    - "pr-merged": a PR was merged
    """

    def __init__(self, config: ReknewConfig, pr_manager: PRManager):
        self.config = config
        self.pr_manager = pr_manager
        self._callbacks: dict[str, list[Callable]] = {}
        self._watched_prs: dict[int, str] = {}  # pr_number → task_id

    def watch_pr(self, pr_number: int, task_id: str) -> None:
        """Start watching a PR for CI and review events."""

    def unwatch_pr(self, pr_number: int) -> None:
        """Stop watching a PR."""

    def on(self, event: str, callback: Callable) -> None:
        """Register callback. Events: ci-completed, review-submitted, pr-merged"""

    async def poll_loop(self) -> None:
        """Polling mode: check all watched PRs periodically.

        Every config.github.poll_interval seconds:
        1. For each watched PR:
           a. Get CI status
           b. If status changed from last check → emit "ci-completed"
           c. Get review comments
           d. If new comments since last check → emit "review-submitted"
        """

    def create_webhook_routes(self) -> FastAPI:
        """Webhook mode: return FastAPI router with POST /webhooks/github.

        Validates webhook signature using config.github.webhook_secret.
        Parses event type from X-GitHub-Event header.
        Emits appropriate events.
        """
```

### 6. reknew/reactions/engine.py

**Purpose:** Config-driven event handler. The brain that connects GitHub events to agent actions.

```python
class ReactionsEngine:
    """Config-driven reactions to GitHub events.

    Pattern from Agent Orchestrator. Three event types with
    automated responses, all configurable via YAML.

    This is the component that closes the automation loop:
    agent produces code → CI runs → if it fails → agent fixes → repeat.
    """

    def __init__(self, config: ReknewConfig, pr_manager: PRManager,
                 spawner: AgentSpawner):
        self.config = config
        self.pr_manager = pr_manager
        self.spawner = spawner
        self._task_retry_counts: dict[str, int] = {}
        self._task_cost_tracking: dict[str, float] = {}

    async def handle_event(self, event_type: str, task_id: str,
                           pr_number: int, repo_name: str) -> None:
        """Route an event to the appropriate handler.

        Args:
            event_type: "ci-completed", "review-submitted"
            task_id: the task that owns this PR
            pr_number: the GitHub PR number
            repo_name: "owner/repo"

        Checks the config to see if this event type has auto-handling enabled.
        If not enabled, logs the event and returns (human handles it).
        """

    async def _handle_ci_failed(self, task_id: str, pr_number: int,
                                 repo_name: str) -> None:
        """Handle CI failure on a PR.

        Steps:
        1. Check retry count against config max retries
        2. If exhausted → close PR with comment, escalate to human, return
        3. Get CI error details from pr_manager
        4. Format error as feedback prompt
        5. Send feedback to agent via adapter
        6. Increment retry count
        7. Log: "[Reactions] Task {id}: CI fix attempt {n}/{max}"
        """

    async def _handle_changes_requested(self, task_id: str, pr_number: int,
                                         repo_name: str) -> None:
        """Handle review comments requesting changes.

        Steps:
        1. Get review comments from pr_manager
        2. Format as feedback prompt
        3. Send to agent via adapter
        4. Start escalation timer (escalate_after from config)
        5. Log: "[Reactions] Task {id}: addressing review comments"
        """

    async def _handle_approved(self, task_id: str, pr_number: int,
                                repo_name: str) -> None:
        """Handle PR approved with green CI.

        If config says auto=true and action=auto-merge:
        → merge the PR via pr_manager.merge_pr()

        If config says auto=false:
        → send notification, wait for human to merge
        """

    def _check_cost_limit(self, task_id: str) -> bool:
        """Check if task has exceeded max_cost for reactions.

        Returns True if under limit, False if exceeded.
        When exceeded, escalate to human.
        """
```

```python
class CIHandler:
    """Parses CI failure logs and formats them for agent consumption.

    Different CI systems produce different log formats.
    This handler normalizes them into a consistent format.
    """

    def parse_github_actions_log(self, log_url: str) -> str:
        """Fetch and parse a GitHub Actions log.

        Downloads the log, extracts the failing step,
        and returns the relevant error output (last 100 lines of the failing step).
        """

    def format_test_failures(self, raw_output: str) -> str:
        """Extract test failure details from pytest/jest/etc output.

        Looks for common patterns:
        - pytest: FAILED lines, assertion errors
        - jest: FAIL lines, expect().toBe() mismatches
        - generic: lines containing "error", "Error", "FAIL", "FAILED"

        Returns: formatted string with just the relevant failures
        """
```

```python
async def run_issue_full_pipeline(repo_name: str, issue_number: int) -> None:
    """Full autonomous pipeline: issue → code → PR → CI → merge.

    Steps 1-9: Same as Phase 2 (fetch, spec, breakdown, parallel agents)
    Step 10: For each completed task:
        a. Push branch to GitHub
        b. Create PR with auto-generated description
        c. Watch PR for CI and review events
    Step 11: Reactions engine handles events:
        - CI fails → re-prompt agent → retry
        - Review comments → re-prompt agent → address
        - Approved + green → notify or auto-merge
    Step 12: Merge worker processes merge queue
    """
```
